Question 1-Setup Git

"Download and install the command line version of the Git version control system if you don't already have it. All code you write from now on (for the entire year!) must be added and committed to a Git repository that you create and manage. For this term I recommend that you make one repository for all your work. Inside this repo/folder each assignment is its own folder. So you'll have one repo and within it folders for HW1, HW2, etc."

Note that this is out of order with how I stumbled around with this. I had only a couple hours of experience with HTML, and some experience with SourceTree, but none with git command line or CSS. This was mostly new to me, much to my chagrin. The blog has an order that makes sense, my attempts to do things did not. First I downloaded git from here. Then I set up git as it prompted me to. It put an icon on my desktop which I now use to get to the command line version of git.

Woo hoo! We have Git! Now, first things first, configure git and add your username and email. I think that this is in order to get the sign in information to line up with the account you made on GitHub so it can sync the information on the local machine to the server. 


                katie@DESKTOP-H160M3S MINGW64 ~/Senior_Sequence/krule1
                $ git config --global user.email "thisIsFakeEmail@mail.wou.edu

                katie@DESKTOP-H160M3S MINGW64 ~/Senior_Sequence/krule1
                $ git config --global user.name "krule14"

Created a repository named Senior_Sequence because I was confused. I created Senior_Sequence repository on the website and then cloned onto my machine using: 


                katie@DESKTOP-H160M3S MINGW64 ~
                $ git clone https://github.com/krule14/Senior_Sequence

Then Created a Repository called Krule14 (which is my Username). I overheard the class talking about needing a repository that was titled with our usernames. So that’s what I did. I made a mistake that I couldn’t fix, more about this later. 


            katie@DESKTOP-H160M3S MINGW64 ~/Senior_Sequence (master)
            $ git clone https://github.com/krule14/krule14.git

Then I demonstrated to myself that I could add and commit from the command line: 


                katie@DESKTOP-H160M3S MINGW64 ~/Senior_Sequence/krule14 (master)
                $ git commit -m "This is my first commit in command line. Hi world"

Which gave me: 


                [master 400d1d1] This is my first commit in command line. Hi world
                1 file changed, 22 insertions(+)
                create mode 100644 index.html

Which I of course verified on the github webpage. It was there after a refresh.

After that, I made the “HW1” file and pushed it to the repository and made sure it was there just like with krule14. Task 1 done! WOOT!
